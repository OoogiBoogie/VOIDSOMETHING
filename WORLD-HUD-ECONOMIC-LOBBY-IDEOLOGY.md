## ğŸ”„ Action Flow (Portal Pattern)

Every action in WORLD HUD should:

1. **Show current state** from relevant hub
2. **Provide one-click navigation** into that hub
3. **Update state** when user takes action in hub
4. **Reflect changes** back in WORLD HUD

**Example: Mission Flow**
1. User sees `[DEFI] Stake into a vault` mission in LeftRail
2. Click â†’ navigates to DeFiHub, pre-filtered to vaults
3. User stakes 100 VOID
4. WORLD HUD updates:
   - Mission progress: 100%
   - Recent rewards: `+50 SIGNAL`
   - PlayerChip: VOID balance decreases
   - TopTicker: `DEFI: Vault TVL +100 VOID`

**Ideology:** WORLD HUD is always in sync with the economy. Actions in hubs flow back.
